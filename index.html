Installing FLEXPART v11 with ECMWF Support: A Complete Technical Guide

This post walks through a full FLEXPART v11 installation for ECMWF meteorological data with NetCDF and Fortran support, including resolving common compilation issues. All components were built from source on a Linux HPC system with user-level access.

⚙️ Installation Structure

All components (GCC, HDF5, NetCDF, ecCodes, etc.) will be installed under this directory tree.

Set these variables in your .bashrc or shell:

🌐 FLEXPART v11 Compilation (ECMWF Mode)

  • eta=no: Use hybrid ECMWF coordinate system
  • ncf=yes: Enable NetCDF support

❌ Common Errors and Fixes

Error Cause Solution
cannot find -leccodes_f90 ecCodes built without Fortran Add -DENABLE_FORTRAN=ON
AEC library not found libaec missing Install libaec and add to CMAKE_PREFIX_PATH
GLIBCXX_3.4.30 not found Old libstdc++ in system Build newer GCC (>=13) and use its lib path
gfortran: command not found GCC built without Fortran Rebuild with --enable-languages=c,c++,fortran

📅 Final Notes

Think of ./configure as the blueprint, make as construction, and make install as moving into your house and wiring the electricity.
You now have a clean, custom-built FLEXPART environment with all dependencies resolved, ready to simulate ECMWF-based transport at scale.

when copy the setting from another PC into the release folder, you need to change the permission of the folder by typing chmod -R 755 ./options and chmod 644./options/RELEASES or any other files.

For multi-core processing, you can use the export OMP_NUM_THREADS=4 to set 4 cores (can set for more) and then with ./FLEXPART command to run FLEXPART with multiple processes.

Sometimes, we will find error of “Program received signal SIGSEGV: Segmentation fault - invalid memory reference.” By typing ulimit -s unlimted, we can solve this problem.

2. RDKit

from rdkit import Chem
from rdkit.Chem import Draw

Define the correct Myosmine SMILES

precursor_smiles = “C1CC(=NC1)C2=CN=CC=C2” # Correct Myosmine (C9H10N2)
precursor_mol = Chem.MolFromSmiles(precursor_smiles)

Define predicted fragment SMILES

fragments_smiles = {
“79.0544”: “C1CC(=NC1)”, # Benzene radical cation C6H6-H+
“104.0495”: “C1=CC=C(C=C1)C=[NH+]”, # Pyridine-benzyl cation
“106.0653”: “C1=CC=C(C=N1)C=[NH+]”, # Methyl-pyridine
“132.0683”: “CCC(=[NH+])C2=CN=CC=C2”, # Pyridine-imidazole
“147.0919”: “C1CC(=NC1)C2=CN=CC=C2” # Correct Myosmine (precursor)
}

Convert SMILES to RDKit molecules

fragment_mols = {mz: Chem.MolFromSmiles(smiles) for mz, smiles in fragments_smiles.items()}

Draw precursor and fragments

Draw.MolsToGridImage(
[precursor_mol] + list(fragment_mols.values()),
molsPerRow=3,
subImgSize=(200, 200),
legends=[“Precursor”] + list(fragments_smiles.keys())
)

Method for Deconvolving MS2 Spectra

Introduction

Mass spectrometry (MS) is a powerful analytical technique used to identify and quantify molecules based on their mass-to-charge ratio (m/z). MS2 spectra, also known as tandem mass spectra, are generated by fragmenting precursor ions and measuring the resulting product ions. Deconvolution of MS2 spectra is a crucial step in mass spectrometry data analysis, helping to separate overlapping signals and identify the true components of complex mixtures.

Understanding MS2 Spectra

MS2 spectra typically consist of:

  • Precursor ion (parent ion)
  • Product ions (fragment ions)
  • Neutral losses
  • Isotopic patterns

The deconvolution process aims to:

  1. Separate overlapping isotopic clusters
  2. Identify charge states
  3. Reconstruct the original precursor mass
  4. Resolve complex mixtures

Common Deconvolution Methods

1. Peak Picking and Centroiding

2. Charge State Deconvolution

3. Isotopic Pattern Matching

Visualization Techniques

1. Raw Spectrum Plot

2. Deconvolved Spectrum

Best Practices

  1. Data Preprocessing

    • Smooth the spectrum to reduce noise
    • Remove baseline
    • Normalize intensities

  2. Parameter Optimization

    • Adjust peak picking thresholds
    • Fine-tune charge state range
    • Optimize isotopic pattern matching parameters

  3. Quality Control

    • Validate deconvolution results
    • Compare with known standards
    • Check for systematic errors

Common Challenges and Solutions

  1. Overlapping Peaks

    • Use advanced peak picking algorithms
    • Implement peak deconvolution techniques
    • Consider using machine learning approaches

  2. Noise Reduction

    • Apply smoothing filters
    • Use signal-to-noise ratio thresholds
    • Implement wavelet transforms

  3. Charge State Determination

    • Use isotopic pattern analysis
    • Consider multiple charge states
    • Implement charge state prediction algorithms

Conclusion

Deconvolution of MS2 spectra is a complex but essential process in mass spectrometry data analysis. By implementing appropriate algorithms and visualization techniques, researchers can extract valuable information from complex mass spectra. The Python examples provided in this post serve as a starting point for developing more sophisticated deconvolution methods tailored to specific research needs.

The work transformed from shapefile to geojson file

2月 23 2023 Tech

Geojson provides several advantages compared to shapefiles:

  1. lightweight, text-based, and easily readable format that can be easily shared, transmitted and used on the web.

  2. It supports a wider range of data types compared to shapefile and can be used across multiple platforms and programming languages.

  3. Additionally, geojson files have smaller file sizes, making them easier to store and process. These benefits make geojson a popular choice for geographic information data storage and exchange.
